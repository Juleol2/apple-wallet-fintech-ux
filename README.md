**Objective**
To solve the fragmentation of financial data by centralizing transaction history and visualizing spending patterns directly within the Apple Wallet interface.
+1

**Methodology & Process**
This project followed a rigorous User-Centric Design process verified by iterative usability testing:


User Research: Conducted interviews to identify pain points in current financial tracking.


Data Visualization: Designed interfaces to render dynamic graphs for spending categories (Food, Entertainment, Health).

Iterative Prototyping: Evolved from low-fidelity paper sketches to high-fidelity Figma prototypes based on A/B testing feedback.

Usability Metrics: Measured task completion time and click-rates to optimize the user flow.

**Key Features Designed**

Unified Dashboard: Aggregates data from multiple bank APIs into a single view.

Interactive Analytics: Interactive charts allowing users to filter expenses by time period (Monthly, Weekly).

Categorization Engine: Visual breakdown of spending habits using intuitive iconography.
